解题报告之P1164小A点菜

发表于 2019-03-23

题目大意：

看原题吧，题目描述很清晰

解题思路：

一个简单的dp问题

设 ：
f[i][j] 为 前 i 道菜，花费 j 时的最大方案数。
a[j] 为 第 i 道菜的 价格
那么 对于 i++时 每次 对于新的 第 i 道菜 有三种情况：

  1. 花费 j < a[i] 此时 这道菜 i 对我们来说没意义，就算只买这一道菜也买不到 -> f[i][j] = f[i-1][j];
  2. 花费 j == a[i] 此时 这道菜 i 相比于 前面 f[i-1][j] 只会多一种情况，就是花费 j 买一道菜 i -> f[i][j] = f[i-1][j]+1
  3. 花费 j > a[i] 此时 -> f[i][j] = f[i-1][j] + f[i-1][j-a[i]];//ps: 这种情况 分两种小情况 选 i这道菜的方案数 与 f[i-1][j-a[i]]是等价的
    不选 这道 i 菜 ，则 方案数 和 f[i-1][j] 是等价的。

都要少时 这部分材料 就被浪费掉了。 把所有浪费的加在一起 当 sum - waste > needthis的时候，就没有必要进行下去了，肯定时无解的。

基础背包问题总结

发表于 2019-03-09 | 更新于 2019-03-23 | 分类于 ACM算法

0-1背包

0-1背包描述的是这么一个问题，你有一个背包最大能负重 m吗，现在有 n 个物品每个物品 i 都有与之对应的 价值 v[i] 和重量 w[i]
问：在物体的重量和不超过你的负重m的情况下，能装的最大价值的物品价值和是多少？这就是典型的0-1背包问题

解题思路：运用dp的思想，对于一个新的物品，只用判断在原来的基础上 选 还是 不选这两种状态。
设 f[i][j] 表示 第 i个 物品 剩余负重为 j 的状态下的最大价值和。v[i] 第i个物品的价值。w[i]第i个物品的重量
状态转移方程为 f[i][j] = max(f[i-1][j],f[i-1][j-w[i]]+v[i])

完全背包

完全背包问题与0-1背包相比，0-1背包的物品i 只有一个，放了之后就没了。而完全背包，每个物品 i 都可以无限次放入

多重背包

多重背包 和 完全背包相比而言，区别就在，完全背包，每个物品 i 可以无限次放入，而多重背包，每个物品i 最多放入s[i]次。

题目大意：

给你一个p行，q列的棋盘，一个骑士棋子(骑士的初始点可以是任意的)问你是否存在一种走法，使得骑士可以把棋盘的所有格都走一遍
骑士每次只能走一个方向只能以‘日’形的走法，如果存在多条路径，则输出字典序最小的一条，如果不存在，则输出impossible。
ps：棋盘的行是以（1，2，3，4。。）的形式，列是以（A，B，C，D。。）的形式

解题思路：

典型的dfs问题，但这题卡时间卡的很死，相比于一般的dfs，要对dfs的顺序做一些优化，首先列举起始点的时候要从列列举
其次dfs的顺序，要从左上角开始列举，这样能够保证找到的第一条路径是字典序最少的，就不需要继续dfs了
最后，用cin，cout输入输出的话，记得用ios：：sync_with_stdio(0),cin.tie(0);来进行优化，否则会超时。

数据结构之线性表

发表于 2019-02-26 | 分类于 ACM算法

前言

本篇文章我们学习线性表，首先我们要知道数据结构分为逻辑结构和物理结构，逻辑结构分为线性结构，集合结构
，树形结构和图形结构四大类。物理结构分为顺序存储结构和链式存储结构。而今天所讲的线性表，自然是属于线性
结构，而根据物理存储的方式不同，线性表又可以分为顺序存储的线性表（顺序表），和链式存储的线性表（链表）

线性表

基本概念

线性表（List）：由零个或多个数据元素组成的有限序列。  
注意：
1.线性表是一个序列。

2.0个元素构成的线性表是空表。

3.线性表中的第一个元素无前驱，最后一个元素无后继，其他元素有且只有一个前驱和后继。

4.线性表是有长度的，其长度就是元素个数，且线性表的元素个数是有限的，也就是说，线性表的长度是有限的。
如果用数学语言来进行定义，可如下：
若将线性表记为（a1,…,ai-1,ai,ai+1,…an）,则表中ai-1领先于ai,ai领先于ai+1,称ai-1是ai的直接前驱元素,ai+1是ai的直接后继元素。

线性表的基本操作

InitList(*L);初始化操作，建立一个空的线性表
ListEmpty(L);判断线性表是否为空，空则返回true，否则返回false
ClearList(*L);将线性表清空
GetElem(L,i,*e);将线性表L中第i哥元素返回给e
LocateElem(L,e);在线性表L中查找元素e，如果查找成功，返回该元素在表中序号，如果查找失败返回0
ListInsert(*L,i,e);在线性表L中的i位置插入元素e
ListDelete(*L,i,*e);删除线性表L中第i个位置上的元素，并用*e返回其值
ListLength();返回线性表L中元素的个数

对于不同的应用，线性表的操作是不同的，上述操作是最基本的
对于实际问题中涉及的关于线性表的更复杂操作，完全可以用这些基本操作的组合来实现。

两种不同线性表

顺序表

顺序表是指顺序存储结构的线性表，指的是用一段地址连续的存储单元依次存储线性表的数据元素。  
顺序表表现在物理内存中，也就是物理上的存储方式，事实上就是在内存中找个初始地址，然后通过
占位的形式，把一定的内存空间给占了，然后把相同数据类型的数据元素依次放在这块空地中。注意，
这块物理内存的地址空间是连续的。   
举 个例子，比如C语言中的基本变量的存储就是连续的存储在内存中的，比如声明一个整数i，在64位
系统中整数i在内存中占8字节，那么系统就会在内存中为这 个整型变量分配一个长度为8个字节的连续
的地址空间，然后把这个i的二进制形式从高地址向低地址存储，长度不足时候，最高位用0补齐。

什么是Ajax？

Ajax 是一种在无需重新加载整个网页的情况下，能够更新部分网页的技术。

Ajax的全称是Asynchronous JavaScript and XML，即异步JavaScript+XML。它并不是新的编程语言，而是几种原有技术的结合体。
通俗的讲呢，我个人的理解是，Ajax就是一种获取和发送数据，客户端与服务器端交互的手段，主要用于前后台的数据交互，数据获取，界面
数据异步更新等它由以下几种技术组合而成，包括：

HTML/XHTML——主要的内容表示语言。
CSS——为XHTML提供文本格式定义。
DOM——对已载入的页面进行动态更新。
XML/json——数据交换格式。
XSLT——将XML转换为XHTML（用CSS修饰样式）。
XMLHttp——用XMLHttpRequest来和服务器进行异步通信，是主要的通信代理。
JavaScript——用来编写Ajax引擎的脚本语言。
实际上，在Ajax解决方案中这些技术都是可用的，不过只有三种是必须的：HTML/XHTML、DOM以及JavaScript。
